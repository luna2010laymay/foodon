# 푸드온(foodon) — 작업 안내

성분(전성분·알레르기·복합성분)으로 포장식품을 검색하는 한국어 React/Vite 앱.
이 파일은 **새 세션이 바로 이어받아 상품을 추가**할 수 있게 정리한 규칙서예요.

## 이 앱의 핵심 작업: "상품 추가"
사용자가 **라벨 사진 + 상품 사진 + 상품명**을 보내면 →
1. 상품 사진 **누끼**(배경 흰색 + 정사각 중앙정렬) 처리 → `public/products/<이름>.jpg`
2. 라벨의 **전성분·영양성분·알레르기(혼입가능)**를 옮겨 적기
3. `src/products.json` 배열에 상품 객체 **추가**
4. `npm run build` 로 빌드 확인
5. **커밋 + 푸시** (아래 배포 규칙 참고)

## 배포 구조 (중요)
- **Vercel이 배포하는 브랜치 = `main`.** 라이브 주소: https://foodon-nine.vercel.app/
- 개발 브랜치 = `claude/github-file-sync-afzgxs`
- 상품이 라이브에 보이려면 **`main`에 반영돼야 함.** 사용자가 "메인에 올려줘/보이게 해줘"라고 하면 main으로 푸시.
- 푸시 후 Vercel이 1~2분 뒤 자동 재배포. 캐시 때문에 안 보이면 Ctrl+Shift+R.

## products.json 스키마
```jsonc
{
  "id": 65,                       // 기존 최대 id + 1
  "name": "오뚜기 토마토 케챂 (300g)", // 이름 뒤에 반드시 (용량) 표기
  "brand": "오뚜기",
  "cat": "조미",                   // 아래 카테고리 목록 중 하나
  "emoji": "🍅",                   // 사진 없을 때 대체용
  "img": "/products/ketchup.jpg",  // 누끼한 파일 경로
  "ing": [                         // 전성분: [성분명, [라벨...]]
    ["토마토페이스트(외국산:미국,칠레,중국 등)", []],
    ["잔탄검", ["additive"]]
  ],
  "contains": ["토마토"],          // 알레르기 유발성분 대표명(있으면)
  "facility": ["대두","밀","우유","계란","쇠고기","조개류","닭고기"], // "이 제품은 ~를 사용한 제품과 같은 시설..." 혼입가능
  "nutrition": {
    "serving": "100 g",
    "kcal": 120,
    "rows": [                      // [항목, "값 단위", "퍼센트%"]
      ["나트륨", "1,040 mg", "52%"],
      ["탄수화물", "29 g", "9%"],
      ["당류", "21 g", "21%"],
      ["지방", "0 g", "0%"],
      ["트랜스지방", "0 g", ""],
      ["포화지방", "0 g", ""],
      ["콜레스테롤", "0 mg", "0%"],
      ["단백질", "1 g", "2%"]
    ]
  },
  "reviews": []
}
```
- **성분 라벨 종류**: `"allergy"`(알레르기 유발), `"additive"`(첨가물), `"animal"`(동물성). 복합성분은 코드가 이름으로 자동 판별(라벨 불필요).
- **복합/다구성 제품**(떡볶이 키트=떡+소스, 라면=면+스프 등)은 `ing` 대신 **`ingGroups`** 사용:
  ```jsonc
  "ingGroups": [ { "name": "찰곤약떡볶이떡", "ing": [["쌀가루(쌀:외국산)", []], ...] }, { "name": "양념", "ing": [...] } ]
  ```
- 카테고리(cat) 사용값: 음료·과자·초콜릿·면류·라면·간편식·두부·묵·유제품·축산·델리·수산가공·농산가공·당류·조미·양념·소스·반찬·떡류·빵·만두 등 (앱이 자동 그룹핑).

## 상품명 규칙
- **모든 상품명 뒤에 용량을 `(80g)` 형식으로** 붙인다. (예: `(300g)`, `(960ml)`, `(120g×5)`, `(2인분)(403.7g)`)
- **풀무원 제품은 이름 앞에 `풀무원 `** 접두어. (오뚜기 등 다른 브랜드도 라벨 표기대로 접두)
- **포장만 다르면 다른 상품**으로 취급(중복 제거하지 말 것).
- 리스트에 없는 상품도 그냥 추가한다(사용자가 보낸 건 다 넣음).

## 영양성분 % 계산 (한국 1일 기준치)
나트륨 2000mg · 탄수화물 324g · 당류 100g · 식이섬유 25g · 단백질 55g · 지방 54g · 포화지방 15g · 콜레스테롤 300mg · 칼슘 700mg · 철 12mg.
`퍼센트 = round(값 / 기준치 * 100)%`. **트랜스지방은 % 없음**(빈 문자열).
라벨에 %가 이미 있으면 그대로 옮겨 적으면 됨.

## 누끼(배경제거) 방법
```bash
python3 scripts/nukki_batch.py --indir <raw폴더> --outdir <out폴더> --method flood --flood-thr 42
```
- Pillow+numpy+scipy 필요(오프라인 flood-fill). 결과: 흰 배경 정사각 JPEG.
- 사용자가 보낸 이미지는 대화 트랜스크립트에서 추출(스크래치패드의 `dump_all.mjs` 참고). 현재 메시지 이미지는 1턴 정도 늦게 잡히기도 함.

## 코드 주의사항
- **`src/IngredientSearchApp.jsx`에는 NUL 바이트가 정확히 4개** 있음(전성분 괄호 처리용 sentinel, 47~49번째 줄). 편집 시 보존해야 함 →
  Python `open(...,'rb')` 바이트 모드로 편집하고 `.count(b'\x00')==4` 확인. products.json엔 NUL 없음(보통 json 편집 OK).
- 상품 데이터는 `src/products.json`, 앱은 거기서 `import PRODUCTS`. 이미지는 `<img loading="lazy">` 지연로딩.

## 커밋/푸시
```bash
git add -A
git pull --no-edit origin <branch>   # 충돌 예방
git commit -q -m "<상품명> 추가"
git push -u origin <branch>          # 네트워크 실패 시 2s→4s→8s→16s 백오프 재시도
```
- 개발은 `claude/github-file-sync-afzgxs`, 라이브 반영은 `main`.
- **API 키(data.go.kr / 식품안전나라)는 절대 커밋 금지**(환경변수로만). 비밀번호처럼 취급.

# 푸드온 배포 가이드 (Vercel)

터미널 없이, 브라우저 클릭만으로 배포합니다.
순서: ① GitHub에 코드 올리기 → ② Vercel에 연결 → ③ 링크 받기

---

## 준비물
- 이 폴더의 파일 전부 (node_modules 폴더는 없어도 됩니다. Vercel이 알아서 설치합니다.)
- GitHub 계정, Vercel 계정 (둘 다 무료)

---

## 1단계 — GitHub에 코드 올리기

1. github.com 접속 → 가입/로그인
2. 오른쪽 위 **+** → **New repository** 클릭
3. Repository name 에 `foodon` 입력 → **Public** 선택 → **Create repository**
4. 다음 화면에서 **uploading an existing file** (파란 글씨) 클릭
5. 이 폴더 안의 파일들을 **드래그해서 업로드**
   - 올릴 것: `index.html`, `package.json`, `vite.config.js`, `.gitignore`, `src` 폴더(안의 파일까지)
   - 올리지 말 것: `node_modules`, `dist` (있어도 무방하지만 불필요)
6. 아래 **Commit changes** 클릭 → 끝

> 팁: `src` 폴더째 드래그하면 안의 파일(main.jsx, IngredientSearchApp.jsx, index.css)이 함께 올라갑니다.

---

## 2단계 — Vercel에 연결

1. vercel.com 접속 → **Continue with GitHub** 로 가입/로그인
2. **Add New… → Project** 클릭
3. 방금 만든 `foodon` 저장소 옆 **Import** 클릭
4. 설정은 **건드리지 않아도 됩니다** (Vercel이 Vite를 자동 인식)
   - Framework Preset: Vite (자동)
   - Build Command: `npm run build` (자동)
   - Output Directory: `dist` (자동)
5. **Deploy** 클릭 → 1~2분 기다리기

---

## 3단계 — 링크 받기

- 배포가 끝나면 `foodon-xxxx.vercel.app` 같은 주소가 생깁니다.
- 그 링크를 복사해서 공유하면, 받은 사람이 바로 앱을 사용할 수 있습니다. (베타/시연용 완료)

---

## 코드를 수정하면?

GitHub에서 파일을 고치면(또는 새로 업로드하면), Vercel이 **자동으로 다시 배포**합니다. 링크는 그대로 유지됩니다.

## 주소를 바꾸고 싶으면?

Vercel 프로젝트 → **Settings → Domains** 에서 이름을 바꾸거나, 가진 도메인을 연결할 수 있습니다.

## 자주 묻는 것
- **돈이 드나요?** 이 정도 사용은 무료입니다.
- **느려요** 처음 열 때 잠깐 느릴 수 있어요. 시연 직전에 한 번 열어두면 매끄럽습니다.
- **빌드 실패가 떠요** 보통 파일 누락입니다. `package.json`과 `src` 폴더(3개 파일)가 다 올라갔는지 확인하세요.
